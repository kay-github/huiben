## 一、产品定义不可改变

这是一个微信小程序，产品名称为 **回本计算器**。

它是一个**极简、单功能、小而美**的工具产品，唯一功能是：

用户输入 **亏损幅度（%）**，系统立即计算并展示 **回本涨幅（%）**。

计算公式：

`回本涨幅 = (1 / (1 - 亏损幅度 / 100) - 1) * 100`

示例：
- 亏损 40%，回本涨幅 66.67%
- 亏损 50%，回本涨幅 100.00%
- 亏损 70%，回本涨幅 233.33%

## 二、你上一版存在的问题，V2 必须修正

### 1. 超出产品边界

你加入了以下内容：
- 数学原理
- 风险提示

这些都不属于本产品范围，必须删除。

本产品不是教育页，不是投资内容页，不是金融知识页。它只是一个高品质计算工具。

### 2. 视觉不够高级，偏模板化

上一版虽然清晰，但整体更像“通用深色金融模板”或“普通 H5 页面”，缺少：
- 克制感
- 精致层次
- 材质细节
- 大厂审美中的高级留白
- 让人记住的视觉记忆点

V2 需要把视觉气质提升到：
- 更克制
- 更统一
- 更贵
- 更像精品工具，而不是信息页面

### 3. 中英混用破坏完整性

上一版出现了：
- Recover Losses Calculator
- QUICK REFERENCE
- LOSS 10%

V2 必须统一语言风格。

要求：
- **只使用中文**
- 不要再混用中英文标题、英文标签、英文说明

### 4. 速查区过重，喧宾夺主

上一版的速查区卡片过大、数量感过强、视觉权重过重，抢走了主功能注意力。

V2 要求：
- 速查区必须更轻
- 更像辅助信息层，而不是第二主模块
- 尽量降低块状感和压迫感
- 视觉重点必须始终是“输入”和“结果”

### 5. 主标题区存在感太弱

上一版中部标题太淡，像占位文字，没有建立品牌气质。

V2 要求：
- 主标题区需要更有品牌感
- 要体现高级、冷静、可信的气质
- 但仍要克制，不能过于花哨或营销化

### 6. 输入区质感不够精致

上一版输入卡片偏钝、偏普通容器，不够细腻。

V2 要求：
- 输入区要更有质感
- 更像高端效率工具中的核心交互区
- 细节上可通过材质、边界、光感、留白、微层次来提升

### 7. 结果区还不够惊艳

上一版的结果数字足够大，但表达方式仍然较常规。

V2 要求：
- 结果数字仍然必须是第一视觉焦点
- 要更有“精致科技感”和“高价值感”
- 可以通过更好的对比、层级、光感、数字布局来增强记忆点
- 但不能炫技，不能影响阅读效率

上一版出现了 `60%`、`90%` 等档位，不符合当前定义。

## 三、V2 设计目标

请把这次设计做成：

- 一眼高级
- 一眼就懂
- 一眼就想用
- 看起来没有广告味、没有金融资讯味、没有表单味

设计方向参考：
- Apple 的克制与留白
- Stripe 的高级渐变控制
- Linear 的精致暗色层次

关键词：
- 极简
- 高级
- 优雅
- 惊艳
- 专业可信
- 安静但有张力

## 四、页面结构要求

只允许设计**一个主页面**，首屏完成主要任务。

页面结构只能包含：
1. 顶部标题区
2. 输入区：亏损幅度（%）
3. 结果区：回本涨幅（%）
4. 轻量速查区
5. 底部轻提示：仅作计算参考，不构成投资建议

禁止新增：
- 数学说明
- 风险教育
- 投资建议说明模块
- Banner
- 图表
- 多页面入口
- 登录注册
- 分享按钮
- 广告位

## 五、交互要求

- 输入后实时计算
- 不要“计算按钮”
- 可保留清空输入按钮
- 速查档位可点击带入输入框
- 输入为空时，有空状态
- 输入大于等于 100% 时，有异常状态
- 所有状态都必须服务于“打开就算、立即出结果”

## 六、视觉执行要求

### 1. 整体风格

- 建议继续采用深色高级基调，但要比上一版更细腻、更统一
- 可以做轻玻璃感、轻磨砂感、轻微环境光感
- 避免粗糙的大黑块和大灰块堆叠
- 避免普通网页卡片感

### 2. 色彩

- 背景建议：石墨黑、冷灰黑、深海蓝黑
- 强调色建议：低饱和电光蓝、青蓝、微紫、暖金中的一种或两种
- 结果数字颜色需要高级、清透、醒目
- 不要大面积使用红绿

### 3. 输入区

- 输入区要有明显但克制的交互感
- 输入数字要大，但不要压过结果区
- 百分号、标签、容器边界要处理得更精致
- 输入卡片不应像普通灰色表单块

### 4. 结果区

- 结果区必须是页面视觉核心
- `66.67%` 这种结果数字应具备高级感和记忆点
- 可增强数字周围的光感、材质反差、空间留白
- 辅助说明文案要更少、更短，不要写成长句解释

### 5. 速查区

- 速查区必须明显弱于主结果区
- 可采用更轻的胶囊、细卡片、列表片段或矩阵方案
- 不要再做成大量厚重黑卡片
- 不要让速查区成为页面第二视觉中心

### 6. 文案风格

统一使用中文。

允许文案：
- 回本计算器
- 亏损后，需要涨多少才能回本
- 亏损幅度
- 回本涨幅
- 仅作计算参考，不构成投资建议

不允许：
- 英文标题
- 英文标签
- 营销文案
- 教育型长文案
- 金融风险内容模块

1. 空状态

要求这些状态：

- 不破坏页面极简气质
